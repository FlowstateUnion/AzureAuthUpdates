# Azure Auth Updates Project

## Purpose
Modernize authentication across Azure Automation runbooks — replace legacy credential patterns with Managed Identity, certificate auth, and Key Vault integration. Includes runtime migration (PS 5.1 → 7.4), operational readiness, and incident response.

## If You Are the Migration Agent

**Read `agent/AGENT-INSTRUCTIONS.md` immediately.** It contains your full operating instructions.

Quick start:
1. Run `agent/skills/initialize-session.ps1` to scan source files and set up tracking
2. For each runbook: analyze → migrate → validate → update progress
3. Your workspace is `runbooks/staging/`. Validated work goes to `runbooks/testing/`.
4. Track everything in `agent/PROGRESS.md` and `agent/progress.json`
5. Never modify `runbooks/source/`. Never move files to `runbooks/completed/`.

Skills available in `agent/skills/`:
- `initialize-session.ps1` — Start here. Scans source, initializes progress.
- `analyze-runbook.ps1` — Analyze a single runbook for migration needs.
- `migrate-runbook.ps1` — Reference guide for applying migration transforms.
- `validate-runbook.ps1` — Syntax check + pattern verification → advances to testing/.
- `update-progress.ps1` — Update progress tracker after each step.

## Project Structure
```
runbooks/               — THE MIGRATION PIPELINE
  source/               ← Human puts original production scripts here (READ-ONLY for agent)
  staging/              ← Agent works here (active workbench)
  testing/              ← Validated scripts ready for human Azure test
  completed/            ← Human moves here after Azure test passes
  exceptions/           ← Scripts that cannot be migrated (with .reason.md)
  PIPELINE.md           — Pipeline rules and folder descriptions

agent/                  — AGENT OPERATING FILES
  AGENT-INSTRUCTIONS.md — Full autonomous operating instructions
  skills/               — Executable skill scripts (analyze, migrate, validate, track)
  progress.json         — Machine-readable progress state
  PROGRESS.md           — Human-readable progress dashboard
  scan-results.csv      — Legacy auth scan output (generated by initialize-session)
  permission-audit.csv  — Permission requirements (generated by initialize-session)

plan/                   — Master plan, phase instructions, rollback playbook, monitoring
strategy/               — Per-pattern migration strategies (01-08)
modules/                — Shared PowerShell module (Contoso.Automation.Auth v1.1)
templates/              — Standardized runbook template
scripts/setup/          — Infrastructure provisioning (MI, Key Vault, runtime env)
scripts/migration/      — Runbook export, scanning, and analysis tools
diagnostics/            — Diagnostic logs and context data
```

## Key Decisions
- Managed Identity is the primary auth method (replaces credentials, client secrets, Run As)
- PnP.PowerShell replaces Microsoft.Online.SharePoint.PowerShell (SPO module)
- EXO v3 with MI replaces credential-based Exchange Online auth
- Target runtime is PowerShell 7.4 (with documented PS 5.1 exceptions)
- All runbooks use the shared Contoso.Automation.Auth module v1.1
- Azure Key Vault for any remaining secrets/certificates
- Module versions are pinned in custom runtime environments
- Child runbooks are migrated before parent runbooks
- Least-privilege permissions audited per runbook (not blanket Sites.FullControl.All)
- Token refresh at 45-min intervals for long-running runbooks

## For the Human

**To start migration:**
1. Complete Phase 0 (infrastructure) — see `plan/PHASE0-SETUP-INSTRUCTIONS.md`
2. Copy your production `.ps1` runbook scripts into `runbooks/source/`
3. Start a new Claude Code session in this folder — the agent reads this CLAUDE.md and begins
4. Check `agent/PROGRESS.md` at any time to see status
5. Review scripts in `runbooks/testing/` and test them in Azure Automation Test pane
6. Move approved scripts to `runbooks/completed/`
7. Publish to Automation Account when ready
